항목별 구현 설명

01관심 행사 북마크

카드 우측 저장 버튼을 누르면 event id를 브라우저에 저장합니다. 상단의 “저장한 행사만” 필터는 저장된 id 목록으로 현재 이벤트 배열을 필터링합니다.

  • 저장소: localStorage.savedEventIds
  • 필요 데이터: event.id
  • 적용 위치: 행사 카드, /events 필터 영역

02최근 본 행사

행사 카드를 클릭하거나 상세 페이지에 진입할 때 event id를 최근 목록 맨 앞에 기록합니다. 기존 리스트 API 응답에서 id를 매칭해 작은 카드로 다시 보여줄 수 있습니다.

  • 저장소: localStorage.recentEventIds
  • 정책: 중복 제거 후 최근 5개 유지
  • 적용 위치: 홈, /events 상단, 상세 추천 영역

03행사 마감 임박 뱃지

행사의 모집 종료일 또는 종료일을 오늘 날짜와 비교해 뱃지를 계산합니다. 리스트와 캘린더에 같은 계산 함수를 공유하면 화면 간 표현이 일관됩니다.

  • 오늘 마감: 남은 일수 0일
  • 3일 남음: 남은 일수 1-3일
  • 이번 주 마감: 남은 일수 4-7일

04캘린더 월간 요약 패널

월간 캘린더 응답을 다시 요청하지 않고 클라이언트에서 집계합니다. 행사 수, 온라인/오프라인 비율, 최빈 태그를 계산해 캘린더 우측 또는 헤더 아래에 배치합니다.

  • 필요 데이터: 월간 이벤트 배열
  • 집계 값: 전체 수, 유형 비율, 태그 빈도
  • 효과: 캘린더 뷰의 정보 밀도 향상

05행사 카드 밀도 전환

같은 데이터를 카드형과 컴팩트형으로 전환합니다. 많은 행사를 빠르게 훑는 사용자는 컴팩트형, 썸네일 중심 탐색은 카드형을 선택할 수 있습니다.

  • 상태: query 또는 localStorage에 viewDensity 저장 가능
  • API 영향: 없음
  • 주의: 모바일에서는 터치 영역 32px 이상 유지

06날짜별 빠른 탐색

“오늘”, “이번 주”, “이번 달”, “다음 달” 칩은 기존 date 필터 또는 query string으로 변환할 수 있습니다. 사용자가 날짜 조건을 직접 입력하지 않아도 빠르게 범위를 좁힙니다.

  • 상태 위치: EventContext의 date 또는 URL query
  • 화면 공유: 리스트와 캘린더 공통 사용
  • 장점: 검색 진입 장벽 감소

07빈 상태 개선

결과 없음 화면에 복구 액션을 제공합니다. “지역 필터 해제”, “온라인 행사만 보기”, “이번 달 전체 보기”처럼 현재 필터 상태에서 바로 실행 가능한 액션을 제안합니다.

  • 필요 데이터: 현재 필터 상태
  • API 영향: 없음
  • 효과: 막힌 화면에서 다음 행동 유도
Backend-light Feature Report

추가 제안: 행사 제보·수정 요청 큐

사용자가 누락된 행사나 잘못된 행사 정보를 직접 제보하게 만드는 기능입니다. 기존 행사 조회 API, 이벤트 모델, 필터 로직은 건드리지 않고 제출용 엔드포인트 하나와 간단한 검수 상태만 추가하면 됩니다.

Low 백엔드 변경 규모: 작음
프론트 체감 가치: 높음

왜 이 기능이 좋은가

Dev Event Web은 큐레이션 서비스라서 행사 데이터의 신선도가 곧 서비스 품질입니다. 제보 기능은 운영자가 직접 찾지 못한 웨비나, 컨퍼런스, 해커톤 정보를 사용자에게서 받을 수 있어 콘텐츠 풀을 늘리는 데 바로 도움이 됩니다.

사용자 가치 내가 아는 개발자 행사를 쉽게 제출하고, 잘못된 날짜나 링크를 신고할 수 있습니다.
운영 가치 행사 수집 채널이 하나 늘어나며, 검수 큐를 기준으로 등록 우선순위를 정할 수 있습니다.
기술 범위 이벤트 조회/검색 API를 수정하지 않고 별도 submit API와 관리자 확인 흐름만 추가합니다.
프론트엔드 행사 제보 버튼, 제출 폼, URL/날짜/주최자 입력, 제출 완료 토스트, 상세 페이지의 “정보 수정 요청” 진입점
백엔드 POST /front/v2/event-submissions 또는 Next API proxy 1개. 입력 검증 후 Slack, 이메일, DB 테이블 중 하나로 전달
데이터 title, url, startDate, endDate, host, category, memo, requesterEmail 정도의 독립 submission payload
변경 회피 기존 이벤트 목록 응답, 캘린더 API, 필터 API, 인증 플로우, 이벤트 상세 모델은 변경하지 않음
