01
Bookmark · localStorage

관심 행사를 별표로 묶어두기

로그인 없이도 동작하는 클라이언트 사이드 북마크. 카드 우상단의 별 아이콘으로 토글하고, /events/bookmarks 라우트에서 모아 봅니다.

  1. EventContext에 bookmarkedIds: Set<number> 추가, localStorage에 mirror.
  2. 카드/상세/캘린더 셀에서 동일한 토글 인터랙션 — 별 채움 + 살짝 튀어오르는 spring 모션.
  3. 토스트로 즉시 피드백, "실행 취소"로 5초간 되돌릴 수 있음.
  4. 나중에 백엔드 동기화가 필요해지면 동일한 인터페이스 뒤에 API만 끼우면 됨.

02
Calendar Export · .ics

내 캘린더에
한 번에 꽂아두기

행사 상세에서 클릭 한 번으로 Apple · Google · Naver · Outlook 어디든 추가. .ics 파일은 클라이언트에서 30줄 정도의 RFC 5545 포맷터로 직접 생성합니다.

  1. 모델의 use_start_date_time_yn === 'N' 이면 자동으로 all-day 이벤트로 처리.
  2. Google Calendar는 calendar.google.com/render?action=TEMPLATE 쿼리 조립만으로 새 탭.
  3. 다중일 행사(end_date_time 까지)와 알람(VALARM) 포함.
  4. 외부 라이브러리 없이 25KB → 0KB. 번들 영향 zero.

03
Notification · Service Worker

놓치지 않게,
30분 전에 톡.

북마크한 행사 시작 N분 전에 브라우저 알림. 푸시 서버 없이 setTimeout + Service Worker의 showNotification 으로 충분합니다.

  1. 최초 북마크 시 권한 요청 → 거절해도 앱은 정상 동작 (graceful degradation).
  2. 알림 시점은 글로벌 설정 1개 · 5분/30분/1시간/1일 전.
  3. SW가 깨어 있을 때 fire, 안 깨어 있어도 다음 방문에 만회 알림.
  4. 같이 작업하면 PWA 매니페스트도 자연스럽게 추가 — "홈 화면에 설치"까지 한 번에.

밤에도 덜 눈부신 행사 탐색.

05
Filter URL · Presets

필터 조합을
링크로 공유.

지금 EventContext 상태는 URL에 반영되지 않아요. 필터를 URLSearchParams 로 직렬화하면 "프론트엔드 + 서울 + 무료" 같은 조합을 한 줄 링크로 동료에게.

  1. 현재 EventContext의 모든 필드(jobGroupList, eventType, location, coast, search, date)를 쿼리스트링에.
  2. "링크 복사" 버튼 한 번이면 끝 — clipboard API.
  3. 자주 쓰는 조합은 별표로 localStorage에 저장 — 사이드바 드롭다운에서 한 번에 적용.
  4. 캘린더 뷰가 이미 ?view=calendar 패턴을 쓰고 있어 자연스럽게 확장 가능.

늘 시야에.

07
Compare · Client State

"둘 중 어디 갈까"를 5초 만에.

카드에서 체크박스로 2~3개 선택 → 하단에 비교 트레이 → 클릭하면 모달. 날짜 · 장소 · 비용 · 주제 · 정원 · 마감을 나란히. 차이 나는 셀은 KTB Blue 하이라이트.

  1. 선택은 sessionStorage에 잠시 보관 → 새로고침해도 유지.
  2. 차이 감지 로직은 단순 row-wise diff — 같으면 묶고, 다르면 강조.
  3. 마감 임박(D−1, D−2)이나 무료 같은 의사결정 단서는 별도 배지.
  4. 모바일에선 카드 스와이프 비교로 자연스럽게.

08
Keyboard · ⌘K Palette

개발자 사이트답게,
모든 게 키보드로.

어디서든 ⌘K 로 명령 팔레트. 검색 · 뷰 전환 · 필터 초기화 · 다음 행사 점프까지 손이 마우스로 옮겨갈 일을 줄이기. 개발자 행사 사이트가 이걸 안 하는 건 좀 직무유기죠.

  1. 단일 글로벌 KeyboardEvent listener — 입력창 포커스 시엔 자동 무시.

  3. 최근 명령은 상단에 — sessionStorage 기반 MRU 캐시.
  4. 접근성: 모달 진입 시 focus trap, 닫을 때 트리거 요소로 복귀.
